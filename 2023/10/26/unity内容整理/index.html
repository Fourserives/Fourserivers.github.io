事件方法(委托)

以一个例子来理解：

当场景生成阳光，玩家点击阳光后更新阳光数量，此时植物卡牌会根据当前阳光数量来刷新状态（有阳光有CD、有阳光无CD、无阳光有CD、无阳光无CD）

其中充当该功能实现的有两个类，在此期间使用了委托:

在 PlayerManager 类中，AddSunNumUpdateActionListener 方法允许其他模块注册一个方法（通过 UnityAction 委托表示）。这个方法会在 PlayerManager 中的 sunUpdateAction 委托被调用时执行

另外一个类的实现该功能的部分代码为：

PlayerManager.Instance.AddSunNumUpdateActionListener(CheckState)意味着 CheckState 方法被注册到了 PlayerManager 中的 sunUpdateAction 委托中。

因此，每当 PlayerManager.Instance.SunNum 的值发生变化，并且 sunUpdateAction 被调用时，CheckState 方法会被执行。

总体流程为：

  1. 在游戏开始时，PlayerManager 的 Awake 方法中会将 sunUpdateAction 委托初始化为空。
  2. 在 Start 方法中，UIPlantCard通过 AddSunNumUpdateActionListener 将 CheckState 方法注册到了 sunUpdateAction 委托中。
  3. 当 PlayerManager.Instance.SunNum 的值发生变化时，PlayerManager 中的 SunNum 的 set 访问器被调用。
  4. 在 set 访问器中，sunUpdateAction() 被调用，触发了 sunUpdateAction 委托。
  5. 因为 CheckState 方法已经被注册到 sunUpdateAction 委托中，所以 CheckState 方法被执行。
