3D动画

Animator组件的属性：

Update Mode指的是动画的刷新模式

Normal：相当于Update，帧率同步，每帧刷新一次

Animate Physics：相当于FixUpdate，物理更新

Unscaled Time：相当于Update，但是这个模式下会忽略当前的时间尺度Time Scale

更加具体的表现可以参照视频：https://www.bilibili.com/video/BV1x44y1e7Ro

Animation动画的属性：

Motion Time表示的是播放的动画特定的帧(0 - 1)

Cycle Offset表示的是提前量(0 - 1)

区别在于：

Motion Time对应的是播放此时特定的帧，0表示初始，1表示结束，如果0.5，则动画此时显示的是整个动画一半位置的表现

Cycle Offset对应的是提前量，0表示没有提前，1表示提前到结束，如果0.5，则动画此时从整个动画一半位置开始播放(相当于动画最开始都是默认从0帧开始，现在变成从0.5开始)

Foot IK对应的讲解详细参考：https://www.bilibili.com/video/BV1v3411e7am

Write Defaults的详解参考：https://www.bilibili.com/video/BV1WL411c7mK

https://www.cnblogs.com/Esfog/p/Unity_WriteDefaults.html https://blog.csdn.net/yjy99yjy999/article/details/82730948

PS：Write Defaults 该属性值在我们做动画或者使用状态机的时候，最好将其关闭！

动画状态

动画状态的切换时，从A动画切换到B动画可以有多条连线：

多条连线也就意味着从A动画可以有不同情况下会切换为B动画，在Inspector窗口下可以看到当前动画连线的有：

PS：如果选择的是某个动画，我们可以看到Transitions下的多条连线，且此时可以调整它们的顺序；而如果选择的是A到B之间的连线，那么此时我们不可以调整它们的顺序

如果从A动画到B动画上会有不同的连线，这时候就要考虑到动画切换有关的内容了

如果我们要从A动画切换到B动画，可以注意到Transitions上的顺序以及后边的内容，后边两个标签分别表示的是：

Solo：勾选上的优先选择

Mute：勾选上的永远不会使用

而多条连线的动画播放使用的顺序就是：

  1. 优先考虑勾选了Solo的几条分支

    a. 如果有多条，那么按照Transitions的顺序，哪个条件先满足了先使用哪条

  2. 其次考虑没有勾选Solo以及Mute的几条分支

    a. 如果有多条，那么按照Transitions的顺序，哪个条件先满足了先使用哪条

动画状态过渡中断

一般会有的运用场景，若身体动画和面部动画是分开的一层，此时如果身体动画或者面部动画切换到对应状态下的动画，但是另外一个动画却还没播放对应状态下要使用的话，此时就需要进行动画状态的过渡或者中断：

面部A → 面部B → 面部C

身体A → 身体B → 身体C

若从A到C动画，此时身体A→B准备到C了，而面部动画刚从A→B，这时候就要开始进行中断，从而达到身体C动画播放的同时面部也是播放的C动画

动画状态过渡中断主要看动画连线部分的Interruption Source标签

  1. Current State

选择该选项后意味着此时动画切换允许被部分/全部动画打断，当我们启用了该选项后，此时下方对应的Ordered Interruption也可以被勾选上，它表示的意思是是否按顺序来允许其动画可以被打断

如果没有勾选上，那么此时该动画可以被任何情况下该动画连接其他动画而被打断

如果勾选上，那么此时动画根据Transitions 下的顺序来被打断

如图，如果此时勾选上了Ordered标签，在Walk → B的切换情况下，此时Walk → B上方的三个动画切换都可以打断它，但是Walk → C的动画切换不可以打断它

  1. Next State

选择该选项表示的是此时动画切换允许被要切换到的动画对应连接的其他动画切换的时候进行打断，以下边例子举例来说：

当Walk → GrilIdle过渡的时候，这时候可以通过GrilIdle → C 或者 GrilIdle → D这两个动画切换来打断，从而变成了Walk → C 或者 Walk → D

当然，如果此时GrilIdle → C / D 的条件都同时触发的话，仍然会按照在Transitions下的动画切换顺序来进行打断

  1. Current State Then Next State
  2. Next State Then Current State

这两个对应的状态过渡中断就是上边两个的结合体，区别就是谁优先级高先考虑谁

在测试动画会发现一个新的问题，就是实际位移和动画表现的位移不同步，所以接下来可以采用Root Motion动画来解决这个问题

Root Motion动画

Root Motion动画一般是自带位移的动画，当角色使用动画控制器绑定对应的动画并使用，启用Apply Root Motion，这时候如果播放动画，那么角色会根据该动画也自动完成一些位移操作

使用Root Motion可以解决脚本编译要实现的效果与动画效果不一致(比如：出现脚底抹油一样的效果)

但是要注意的是，使用Root Motion会出现一些其他问题：

走直线的动画会发生偏移，走路可能会让脚陷下去一些，使用混合树时表现跟脚本要实现的效果不一致等等

这是因为Root Motion动画与其他动画的区别是：

动画文件会直接修改每一帧里游戏对象的坐标值和角度值，而root motion则通过 相对位移和转角 来移动游戏对象

同样的，如果我们在当前绑定了Root Motion动画的对象上挂载了包含有OnAnimatorMove()方法的脚本，此时的Apply Root Motion便不再被使用，而是把控制权交由该方法来实现：

关于Root Motion机制的讲解可以参考：https://www.bilibili.com/video/BV1kZ4y1B7Tc

Bake Into Pose：

如果勾选了，其表示不要将骨骼节点的旋转当做root motion的一部分来处理，而是将其当做普通动画进行处理

简而言之就是让骨骼不会跟随动画来进行移动，从而导致整体动画产生的位移发生偏移

总结：要不要勾选这个选项取决于我们需不需要动画来驱动游戏对象的旋转，而至于能不能勾选，则要参考后边的红色或者绿色提示

Based Upon：

这里指游戏对象在动画开始时对准的方向是哪里

其中默认的Body Orientation指的是让游戏对象在动画开始时对准根骨骼节点的指向方向，一般来说不准

而Original指的是让游戏对象在动画开始时对准动画最初的方向，一般来说使用这个

  • Root Transform Rotation(Y)

同理，另外一个对应的就是X和Z轴方向上的root motion处理

PS：虽然可以在Root Motion内通过Unity本身自带的功能来调整，但是在实际应用中最好还是使用OnAnimatorMove()方法来调整

根据动画在不同方向上的位移速度，比如Vector3对应的三个方向来让其自动计算匝值，以图中举例，在使用这三个动画，为其自动生成出来的范围就是 -1.2826 —— 0.93

而另外一个Adjuist Time Scale是来解决不同动画的播放速度问题的，如果遇到动画播放效果不一致的话，就可以通过它来为我们自动计算动画播放速度，从而达到动画一致的表现效果

PS：如果要让不同模型的角色使用同个状态机的话，会因为模型骨骼不一致，比如大体型人模与小体型人模的骨骼不同，本质上骨骼scale也不同，而Root Motion动画的位移是与scale有关的，所以会导致同样配置下动作表现会完全不同（比如直线动画走，一个快一个慢），这时候就要通过/= animator.humanScale的方式来获得真正意义上的相同速度

eg：在速度speed都为1的情况下，第一个模型的scale如果是100%，那么此时动画的位移速度也是1:1的情况，而如果第二个模型的scale是68%的话，那么此时动画的位移速度如果要一致的情况下，就要 1 / 0.68 得到真正相同的速度

Root Motion结合Rigid Body完成自定义速度

在前边我们通过Root Motion动画实现了动画与实际速度不一致(比如：脚底打滑)的问题，然后我们通过Root Motion动画来自己操控速度，这与实际开发并不符合，所以我们仍然要用回OnAnimationMove()方法来自己调控动画，而Root Motion只拿来解决动画与实际不一致的问题就行

这时候就要通过Rigid Body组件来完成我们的自定义移动需求了

为模型添加了Rigid Body组件后，由于接下来要通过它来让角色进行移动等操作，这通常都是建立在每一帧调用一次对应的操作，所以这时要回到Animator组件上并且把Update Mode选择为Animate Physics，同时Rigid Body组件也要相应的锁定：

Layer完成多动画组合

**Weight**表示的是该层级动画的权重【0 - 1】，0就表示不起作用

**Mask**表示的是蒙版，作用是可以选择对应模型，在当前层级播放的动画对哪个部位会产生影响

绿色表示受影响，红色表示不受影响，这里使用的是Humanoid选项，如果要更细致一些的可以选择下边的Transform选项，可以精确到骨骼

**Blending**属性则有两个选项，分别对应了 覆盖 和 添加

覆盖指的是当前层级的动画覆盖上一层级的动画，添加指的是当前层级的动画和上一级动画混合起来

  • 覆盖

    eg：角色持不同武器；角色表情控制等等，通常通过覆盖的方式来进行

  • 添加

    eg：角色疲惫状态；异常状态等，通常通过添加的方式来进行混合

PS：动画层级比较多的情况下，从上往下对应的下标为0-n，跟数组下标一样

通常情况下，可以通过animator.GetLayerIndex("层级名字")来获得该层级的序号，然后再后续通过像animater.SetLayerWeight(层级序号，数值)来更改层级的权重值等等

小知识：Mathf.Clamp(变量名，数值最左值，数值最右值)，表示把变量的值限制在一个区间内

**Sync**属性表示的是同步，意思就是两个层级是同步的，这里的应用可以详细参考：https://www.bilibili.com/video/BV1AF411M7eK

同步要注意一个问题就是，相同的动画片段可能播放的时长是不一样的，这会不会导致动画播放不一致？其实不会，因为unity会把使用同步的层级与被同步的对应的动画进行缩放，从而达到一样长的效果
