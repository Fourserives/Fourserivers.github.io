买卖股票的最佳时机

题目：https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/

解题思路：

1、dp数组以及下标含义

dp[i ] [0]：持有股最大值

dp[i ] [1]：无持股最大值

2、递推公式

由于dp[i ] [0]表示持有股所得现金最大值，所以dp[i ] [0]最大值是在两种情况下取：

第一种情况：dp[i-1] [0]也就是之前就买入股的现金

第二种情况：- prices[i]也就是刚买入股的现金

由于dp[i ] [1]表示无持股所得现金最大值，所以dp[i ] [1]最大值是在两种情况下取：

第一种情况：dp[i-1] [1]也就是之前已经把股票售出(所以是无持股)的现金

第二种情况：dp[i ] [0]+prices[i]也就是刚售出股的现金

综上所述，递推公式即：

dp[i] [1] = max(dp[i-1] [1]，dp[i ] [0]+prices[i])

其中，dp[i ] [0] = max(dp[i-1] [0]，-prices[i])

3、初始化

由递推公式可知，初始要从dp[0 ] [0]，dp[0 ] [1]开始，所以初始化为：

dp[0 ] [0] = -prices[0] ， 由于dp[0 ] [0]表示持有股且它为首位，所以只能取第一天就买入股消费的现金，因为只有这样才满足我们对dp[i] [0]的定义！

dp[0 ] [1] = 0，由于dp[0 ] [1]表示无持股且它为首位，所以取第一天没买入股，也就是0

遍历第一列下边(dp[ i] [0])都为1，只有一种方式能到达

1、dp数组以及下标的含义

dp[i]：小偷在第i个房间内能够取得的最高金额dp[i]

2、递推公式

由题目可以推测，假设以第三间房间来算小偷的最高金额，则有以下两种情况：

第一种，小偷进入第一间房间偷取金额dp[0]，间隔一间后偷取本间房间的金额nums[2]

第二种，小偷进入第二间房间偷取金额dp[1]，第三间为间隔所以无法偷取

所以第三间的金额计算就是 max(dp[0]，(dp[1]+nums[2]))

再以第四间房间来算，则两种情况分别是第三间的最高金额dp[2]与第二间的最高金额dp[1]加上相隔一间后本间金额nums[3]

综上所述，可以得到递推公式为：

dp[i] = max(dp[i-1]，(dp[i-2]+nums[i]))

3、初始化

由于dp数组的意义，所以第一间房间金额必然是dp[0] = nums[0]

而第二间房间比较特殊，由于递推公式需要有i-2，如果直接代入公式会导致数组下标为负数，不合法，结合题目情况，有：

dp[1] = max(dp[0],nums[1])

4、遍历顺序

从左到右

5、打印输出

最长递增子序列

题目：https://leetcode.cn/problems/longest-increasing-subsequence/

解题思路：

1、dp数组以及下标含义

dp[i]：以nums[i]为尾的最长递增子序列的长度

2、递推公式

以nums = [10,9,2,5,3,7,101,18]为例

当i = 1时，dp[i] 表示以 nums[1]为尾的最长递增子序列[9]，dp[1] = 1

当i = 2时，dp[i] 表示以nums[2]为尾的最长递增子序列[2]，dp[2] = 1

当i = 3时，dp[i] 表示以nums[3]为尾的最长递增子序列[2，5]，dp[3] = 2

…

位置i的最长升序子序列等于j从0到i-1各个位置的最长升序子序列 + 1 的最大值。

所以：if (nums[i] > nums[j])时，必有 dp[i] = max(dp[i], dp[j] + 1)

综上可以知道，递推公式为：

dp[i] = max(dp[i]，dp[j]+1); （其中j ∈ [0，i-1]）

3、初始化

由于dp数组的含义，每一个i对应的dp[i]（即最长递增子序列）起始大小至少都是1

4、遍历顺序

dp[i]：从左到右

dp[j]：任意

5、打印输出

分割等和子集对应的背包问题相当于：给我们一个容器，问我们能否装满这个容器

分析：

  • 背包的体积为sum / 2
  • 背包要放入的商品（集合里的元素）重量为 元素的数值，价值也为元素的数值
  • 背包如果正好装满，说明找到了总和为 sum / 2 的子集
  • 背包中每一个元素是不可重复放入（01背包的前提）

动规五部曲：

  1. 确定dp数组的下标以及含义

    dp[j] 表示装满重量为j的背包 能够放入的最大重量（最大价值）dp[j] （本题里边每一个元素的值既是重量也是价值）

    所以当dp[target] = target ，也就是装满重量为目标值target的背包所能放入的最大重量就等于目标值target的时候，返回true，反之false

    也有装不满的情况： [1,5,3,7] dp[7]意味着没有装满的情况，因为此时dp[7]的最大重量只能是6，但是dp[6]则表示可以装满

  2. 递推公式

​ dp[j] = max(dp[j], dp[j-nums[i]] + nums[i] ) 也就是01背包的式子

  1. 初始化

    从dp[j]的定义来看，首先dp[0]一定是0。

    如果题目给的价值都是正整数那么非0下标都初始化为0就可以了，如果题目给的价值有负数，那么非0下标就要初始化为负无穷。

    这样才能让dp数组在递推的过程中取得最大的价值，而不是被初始值覆盖了。

    本题题目中 只包含正整数的非空数组，所以非0下标的元素初始化为0就可以了

  2. 确定遍历顺序

    采用滚动一维数组的dp数组，所以按照先遍历物品数量再遍历物品重量的顺序

    物品重量的遍历为什么要倒序可以参考讲解：https://programmercarl.com/%E8%83%8C%E5%8C%85%E7%90%86%E8%AE%BA%E5%9F%BA%E7%A1%8001%E8%83%8C%E5%8C%85-2.html#%E7%AE%97%E6%B3%95%E5%85%AC%E5%BC%80%E8%AF%BE

最后一块石头的重量II

题目链接：https://leetcode.cn/problems/last-stone-weight-ii/description/

最后一块石头的重量II对应的背包问题相当于：给我们一个容量的背包，它最多能够装多少

该题本质上就是尽量让石头分成重量相同的两堆，相撞之后剩下的石头最小

动规五部曲：

  1. 确定dp数组以及下标的含义

​ **dp[j]表示容量（这里说容量更形象，其实就是重量）为j的背包，最多可以背最大重量为dp[j]**。

​ 可以回忆一下01背包中，dp[j]的含义，容量为j的背包，最多可以装的价值为 dp[j]。

​ 相对于 01背包，本题中，石头的重量是 stones[i]，石头的价值也是 stones[i] ，所以 “最多可以装的价值为 dp[j]” == “最多可以背的重 量为dp[j]”

  1. 确定递推公式

​ 依旧是：dp[j] = max(dp[j], dp[j - stones[i]] + stones[i]);

目标和对应的背包问题相当于：给我们一个容器，问装满这个容器有多少种方法

分析：

把集合分为两个子集，一个代表正数集left，一个代表负数集right，那么有：
left + right = sum (总和)

PS：right虽然表示负数集合，但是意思是该里边的数值之和带上负号才是负数，所以此时可以先计算总和(相当于取绝对值)

题目意思可以表示为：
left - right = target（目标值）
综上可以得到式子：
left = (target + sum) / 2 其中target和sum都是已知的值，所以可以求得left

此时问题就转化为在集合nums中找出和为left的组合
