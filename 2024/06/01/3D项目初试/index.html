3D建模渲染管线

若素材因为渲染问题没有正确显示的情况下，要初始化或导入对应版本的渲染URP管线，然后在Project Setting → Graphics 以及 Quality下进行绑定对应的渲染URP

然后在如图的内容下进行使用，从而让建模重新被渲染

具体的渲染内容，针对物品不同或内容不同可能需要进行不同的调整，该界面下如何使用到时候可以在遇到问题的情况下再去了解不同选择的作用，在对人物建模重新渲染这块，通过勾选Material Upgrade然后进行覆盖处理就能让人物建模重新显示出来了

天空盒的使用与切换

在Window → Rendering → Lighting打开可以让我们更改的页面

在Environment选项下切换天空盒

如果为了视角效果的逼真，通常阴影部分还可以添加多段层，从而来达到不同距离下看到的影子效果各不一样：

这块了解即可

创建光线

以上选项对应的内容参考Unity文档手册即可

自己调整成需要的光线后点击Generate Lighting完成光线的编辑

接下来是环境光的设置，环境光源的类型可分为 天空盒、渐变和颜色

其中天空盒是默认设置，其光源要比较自然；渐变则是一种环境光源，利用渐变来表现天空、空气、地面三种光源的颜色，可以创造出具有强烈冲击力的场景；色彩以单一颜色来表达环境光，因此可以创造出非常生动的印象

同时如果因为有天空盒，天空盒的存在也会影响环境光，导致照射的环境光的色彩不一样，这也可以进行调整，比如调整成默认的环境光，也支持自己调整对应颜色：

这部分环境光的内容也可以参考：https://styly.cc/tips/unity-lighting/

详细讲解也可以参考：https://www.youtube.com/watch?v=VnG2gOKV9dw

一些3D快捷键使用技巧

第一个是物品的摆放，一般情况下场景地图不是平坦的而是凹凸的，当我们新创建一个物品或者拖拽一个预制体进行摆放的时候，很难准确地把物品完整地与当前摆放角度摆放界面放置完整下去，这时候就可以通过**V键**来通过顶点快捷置入当前鼠标对应的地面

需要注意的是，该快捷键的摆放遇到凹凸不平，崎岖位置进行摆放，可能会因为顶点的偏移导致很难精准地放置到我们想要放到的斜面位置上

不过也可以通过ctrl ＋ shift拖动物体来与放置的位置进行缝合

第二个就是摄像机地快捷设置，在3D搭建场景里边，为了让Game窗口能够在一个比较符合我们的角度上进行摆放，通常可以在Scene先切换到一个合适地视角里，然后通过选中相机，快捷键Ctrl+shift＋f进行设置，这样相机的视角就是我们当前Scene下的视角了

地图场景

场景快速搭建(polybrush)

导入成功后就可以看到多了一个Tool选项，选择PloyBrush即可打开对应的窗口，通过在窗口中进行场景编辑

通常可以直接基于这两个右键Material创建出对应的材质球，这样就可以继续通过Polybrush工具进行绘制了

polybrush提供了五个小功能，这些功能可以快速进行场景的搭建，具体的运用可以参考：https://blog.csdn.net/qq_57250692/article/details/132864343

https://blog.csdn.net/weixin_50114337/article/details/128314370

场景地图扩大(probuilder)

导入成功后也在Tool下启动ProBuilder的编辑页面，通过probuilder窗口来进行场景制作

通常是左键选择对应的内容，然后再左边的Shape Tool下选择模板，同时可以调整该场景(物体)的大小以及网格数量(1*1)

如果需要细微调动地图位置移动可以了解下ProGrids

然后是地图(一般是用Plane图形)创建好要为其加上颜色，相当于陆地颜色，则要创建一个Material的材质球，然后把材质球拖动过去，进行自定义调色即可

最后是基础地形场景构建可以参考学习的内容：https://blog.csdn.net/WGYHAPPY/article/details/117851090

地图烘焙

首先还是要先下载对应的插件AI Navigation，然后便可以在Window → AI → Navigation打开对应的窗口

该圆柱就是表示人物的大小与宽展，以及能够爬坡的角度等，然后是对当前主场景或者说主地形进行设置，让其选为可行区域(Walkable)或者不可行区域(Not Walkable)

然后通过在Bake选项下进行设置应用

这样就能在场景里看到哪部分区域是可行区域或者不可行区域了(有浅蓝色覆盖就是可行区域)，然后也要对场景里的一些物体，比如树木、巨石、城楼等实际物体的Navigation也进行调整，原理也和上边的操作一样，设置区域完成后也要记得按Bake进行运用，最后可以观察到：

PS：该方法主要是用来设置地图场景默认本身就不能移动的物体，所以通过Navigation里边直接设置可以更加方便，但是如果我们只要让单个物品设置为障碍物不能被经过呢？

这时候可以为单独的物体添加一个障碍物的组件Nav Mesh Obstacle，通过这种方式设置的障碍物/碰撞体本身可以允许被移动，其移动的时候不可行区域也会跟着移动

以上两种方式分别对应了场景物体里边静态/不可移动以及动态/可被移动的设置，当前者物体被移动的时候，其不可行区域依然保持不变，而后者则可以跟着物体被移动

诸如此类下显示的绿色范围就是不可行区域的范围大小了，而由于默认范围大小没有调整，就是按照人物的宽展来的，所以还要对人物的宽展与大小进行设置：

为人物添加一个新的组件Nav Mesh Agent

同时在调整完成后还要对Navigation对应的Bake数值上也进行更改，其大小与宽展也要同人物该组件绑定的数值保持一致！
